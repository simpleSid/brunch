# Современный стек для лендинга кафе

## Цель проекта

Сделать быстрый, красивый и современный лендинг для кафе с акцентом на атмосферу, меню, интерьер, отзывы, адрес и быстрый контакт / бронирование.

Лендинг должен быть:

- визуально премиальным;
- быстрым на мобильных устройствах;
- удобным для SEO;
- простым в поддержке;
- готовым к добавлению меню, акций, фото и контактной формы.

---

## Рекомендуемый стек

```txt
Astro → TypeScript → Tailwind CSS → GSAP / Motion → Sanity CMS → Vercel / Cloudflare Pages
```

---

## 1. Дизайн

### Дизайн

**Для чего:**

- дизайн-макет лендинга;
- подбор визуального стиля;
- работа с сеткой, цветами и типографикой;
- подготовка UI-компонентов;
- адаптивы для desktop / tablet / mobile.

**Что подготовить в Дизайн:**

- Hero-блок;
- блок меню;
- карточки блюд;
- галерею интерьера;
- отзывы;
- блок адреса и контактов;
- финальный CTA.

---

## 2. Frontend

### Astro

**Почему Astro:**

- отлично подходит для лендингов и контентных сайтов;
- быстро загружает страницы;
- отправляет минимум JavaScript в браузер;
- хорошо подходит для SEO;
- проще, чем Next.js, если не нужен личный кабинет, корзина или сложная логика.

**Использовать для:**

- структуры страниц;
- секций лендинга;
- статической генерации;
- SEO-разметки;
- быстрой загрузки сайта.

---

## 3. Язык

### TypeScript

**Для чего:**

- меньше ошибок в коде;
- удобнее поддерживать проект;
- проще работать с данными меню, отзывов, карточек и CMS.

---

## 4. Стили

### Tailwind CSS

**Почему Tailwind:**

- быстро собирать кастомный дизайн;
- удобно делать адаптив;
- легко поддерживать единый стиль;
- хорошо подходит для современных лендингов.

**Использовать для:**

- отступов;
- сеток;
- типографики;
- карточек блюд;
- кнопок;
- адаптивной верстки;
- hover-эффектов.

---

## 5. Анимации

### GSAP

**Для чего:**

- плавные появления блоков;
- scroll-анимации;
- параллакс;
- анимации hero-секции;
- премиальные микровзаимодействия.

**Где использовать:**

- плавное появление hero-текста;
- анимация карточек меню;
- движение фото интерьера при скролле;
- мягкий reveal отзывов;
- финальный CTA.

### Альтернатива: Motion / Framer Motion

Подходит, если проект будет на React-компонентах. Для Astro можно использовать только там, где действительно нужны интерактивные островки.

---

## 6. Контент

### Sanity CMS

**Для чего:**

- редактировать меню без изменения кода;
- добавлять новые блюда;
- менять цены;
- обновлять акции;
- редактировать часы работы;
- загружать фото.

**Что можно хранить в CMS:**

- блюда;
- категории меню;
- цены;
- описания блюд;
- отзывы;
- галерею интерьера;
- адрес;
- часы работы;
- ссылки на соцсети;
- акции и специальные предложения.

### Упрощённый вариант

Если CMS пока не нужна, можно начать с локальных файлов:

```txt
/src/data/menu.json
/src/data/reviews.json
/src/data/gallery.json
```

Такой вариант проще и быстрее для первого запуска.

### Рекомендации

Для кафе визуал очень важен, поэтому нужно заранее продумать работу с изображениями.

**Использовать:**

- Tally;

- время;
- количество гостей;

## 10. SEO

### Обязательно добавить

- title и description для страницы;
- Open Graph для красивой ссылки в соцсетях;
- favicon;
- sitemap.xml;
- robots.txt;

- оптимизированные заголовки H1 / H2 / H3.

### Пример SEO-фокуса

```txt

Завтраки в [городе]
```

## 11. Хостинг

### Vercel

Хороший вариант для быстрого запуска frontend-проекта.

**Плюсы:**

- простой деплой;
- автоматическая сборка из GitHub;
- preview-ссылки;
- хорошая скорость.

### Cloudflare Pages

Тоже отличный вариант для статичного сайта.

**Плюсы:**

- быстрый CDN;
- часто дешевле;
- хорошо подходит для статичных лендингов;
- можно использовать вместе с Cloudflare DNS.

## 13. Структура лендинга

Рекомендуемая структура страницы:

1. **Hero**  
   Атмосферное фото или видео, название кафе, короткий оффер, кнопки “Посмотреть меню” и “Забронировать”.

2. **Зачем прийти**  
   Коротко объяснить, чем кафе отличается: завтраки, кофе, десерты, интерьер, атмосфера.

3. **Хиты меню**  
   4–8 популярных блюд или напитков с фото, ценами и коротким описанием.

4. **Интерьер и атмосфера**  
   Галерея фото, видео, детали пространства.

5. **Для кого кафе**  
   Для завтраков, встреч, работы с ноутбуком, свиданий, семейных выходных.

6. **Отзывы / доказательства**  
   Отзывы гостей, рейтинг, фото из соцсетей.

7. **Адрес, карта, часы работы**  
   Контакты, карта, маршрут, режим работы.

8. **Финальный CTA**  
   Повторная кнопка бронирования или переход в мессенджер.

## 15. Минимальный стек для быстрого запуска

Если нужно сделать быстро и без лишней сложности:

```txt
Astro + TypeScript + Tailwind CSS + JSON-файлы + Vercel
```

Подходит, если:

- меню редко меняется;
- сайт нужен быстро;
- нет отдельного администратора;
- контент можно обновлять через код.

---

## 16. Премиальный стек

Если нужен более дорогой, визуально сильный и удобный в поддержке вариант:

```txt
Astro + TypeScript + Tailwind CSS + GSAP + Sanity CMS + Cloudinary + Vercel
```

Подходит, если:

- будет много фото;
- меню часто обновляется;
- нужны анимации;
- владелец кафе хочет сам менять контент;
- сайт должен выглядеть как премиальный брендовый лендинг.

---

## Итоговая рекомендация

Для лендинга кафе лучше всего выбрать:

```txt
Astro + TypeScript + Tailwind CSS + GSAP + Sanity CMS + Vercel
```

Если проект нужно запустить максимально быстро, начать можно без CMS:

```txt
Astro + TypeScript + Tailwind CSS + локальные JSON-файлы + Vercel
```

А CMS, бронирование и более сложные интеграции можно добавить позже.
